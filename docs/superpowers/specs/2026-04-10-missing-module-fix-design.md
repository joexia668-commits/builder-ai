# Missing Module Fix Design

**Date:** 2026-04-10  
**Status:** Approved

## Problem

When the AI engineer generates multi-file code, it occasionally imports from local file paths that were never defined in the architect's scaffold (e.g., `import { formatDate } from '/utils/format.js'`). The `extractMultiFileCode` function only validates that scaffold-expected files are present — it does not check whether imports *inside* those files reference non-existent paths. At render time, Sandpack crashes with "Could not find module in path" and shows a blank preview with an unreadable error.

Reproduced at V6 generation in a task management project.

## Goals

1. Reduce the frequency of hallucinated local imports (prompt fix).
2. Detect any that slip through and surface a clear, actionable error to the user.
3. Prevent a white-screen Sandpack crash as a last resort.

## Non-Goals

- Automatically regenerating the missing file (too complex, out of scope).
- Fixing other categories of engineer hallucination (wrong package names, bad JSX, etc.).

## Design

### Layer 1 — Prompt Reinforcement (`lib/generate-prompts.ts`)

Add a `【本地文件导入限制】` rule to `getMultiFileEngineerPrompt`, at the same weight level as the existing `【严禁包限制】` block:

```
【本地文件导入限制】
只允许 import 以下本地路径：
- deps 列表中明确列出的文件路径
- /supabaseClient.js
禁止 import 任何未在 deps 中出现的本地路径（如 /utils/format.js、/helpers/xxx.js 等）。
如果需要工具函数，必须在当前文件内自己实现，不得假设存在其他文件。
```

Placement: inserted immediately after the `【严禁包限制】` block, before the design notes section.

### Layer 2 — Post-Generation Detection (`lib/extract-code.ts`)

New exported pure function:

```typescript
/**
 * Scan all generated files for imports of local paths ('/...') that are not
 * present in the files map. Returns a deduplicated list of missing paths.
 * Whitelist: /supabaseClient.js is always considered present.
 */
export function findMissingLocalImports(
  files: Record<string, string>
): string[]
```

Implementation:
- Regex: `/from\s+['"](\/.+?)['"]/g` on each file's code
- Skip paths already in `Object.keys(files)` and `/supabaseClient.js`
- Return deduplicated array of missing paths

**Integration point:** `chat-area.tsx`, in the `files_complete` SSE handler, after all layers are merged into `allFiles` and before calling `onFilesGenerated`. If `findMissingLocalImports(allFiles)` returns a non-empty list:

- Set a generation error state with message:  
  `"AI 生成的代码引用了未创建的文件：[paths]。请重新生成。"`
- Still call `onFilesGenerated` so the Sandpack stub (Layer 3) renders something rather than nothing.

### Layer 3 — Sandpack Stub Injection (`lib/sandpack-config.ts`)

Inside `buildSandpackConfig`, after building `sandpackFiles` from `userFiles`, call `findMissingLocalImports(userFiles)`. For each missing path, inject:

```js
// Auto-stub: <path> was not generated by AI
export default new Proxy({}, {
  get(_, key) {
    console.warn(`[Builder AI] Missing module stub: <path> — "${String(key)}" called on missing module`);
    return () => null;
  }
});
```

`Proxy` is used because the exported names are unknown at stub-generation time. Sandpack's module system maps named imports to `default.name`, so the Proxy intercepts any named access and returns a no-op function rather than throwing.

The stub is marked `hidden: true` so it doesn't appear in the code editor.

## File Changes

| File | Change |
|------|--------|
| `lib/generate-prompts.ts` | Add `【本地文件导入限制】` rule to `getMultiFileEngineerPrompt` |
| `lib/extract-code.ts` | Add `findMissingLocalImports(files)` function |
| `lib/sandpack-config.ts` | Call `findMissingLocalImports` and inject stubs for missing paths |
| `components/workspace/chat-area.tsx` | Call `findMissingLocalImports` after layer merge; show error if non-empty |

## Error Message (UX)

Displayed in chat as a generation error (same UI as existing `errorCode` errors):

> AI 生成的代码引用了未创建的文件：`/utils/format.js`。请重新生成。

## Testing

- Unit test `findMissingLocalImports`: files with no missing imports, single missing import, multiple missing imports, `/supabaseClient.js` correctly whitelisted.
- Unit test `buildSandpackConfig`: verify stub is injected when a file imports a missing path.
- Existing `extractMultiFileCode` tests unchanged.
