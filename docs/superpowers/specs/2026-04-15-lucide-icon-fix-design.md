# Lucide 图标名自动修复设计

> **For agentic workers:** Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this spec.

**Goal:** 在后处理阶段自动检测并修复 LLM 生成的 lucide-react 无效图标名（如 `CalculatorIcon` → `Calculator`），在代码送入 Sandpack 之前消除运行时 `ReferenceError`。

**方案选型:** 静态替换（不调 LLM）。从 lucide-react 包提取完整图标列表，通过后缀去除 + 编辑距离匹配找到正确名称，直接字符串替换。

---

## 1. 图标列表生成

### 1.1 生成脚本

新增 `scripts/generate-lucide-names.ts`：
- 从 `lucide-react` 包导入所有导出
- 过滤 PascalCase 名称（`/^[A-Z]/`），排除工具函数（`createLucideIcon` 等）
- 输出到 `lib/lucide-icon-names.ts`

### 1.2 生成的文件

`lib/lucide-icon-names.ts`：

```typescript
// Auto-generated by scripts/generate-lucide-names.ts — do not edit manually
export const LUCIDE_ICON_NAMES: ReadonlySet<string> = new Set([
  "Activity", "Airplay", "AlertCircle", /* ... ~1500 */
]);
```

### 1.3 运行方式

`package.json` 新增 script：
```json
"gen:icons": "tsx scripts/generate-lucide-names.ts"
```

手动执行 `npm run gen:icons`，仅在 lucide-react 版本更新时需要重新运行。生成的文件提交到 git。

## 2. 检测函数

### 2.1 类型定义

`lib/types.ts` 新增：

```typescript
export interface LucideIconFix {
  readonly filePath: string;
  readonly original: string;     // "CalculatorIcon"
  readonly replacement: string;  // "Calculator"
}
```

### 2.2 checkUndefinedLucideIcons

新增于 `lib/extract-code.ts`：

```typescript
function checkUndefinedLucideIcons(
  files: Readonly<Record<string, string>>
): LucideIconFix[]
```

**逻辑：**

1. 遍历所有文件，用正则提取 `import { X, Y, Z } from "lucide-react"` 中的每个名称
2. 对每个名称，检查是否存在于 `LUCIDE_ICON_NAMES`
3. 如果不存在，按优先级尝试匹配：
   - 去掉 `Icon` / `Outline` / `Solid` 后缀后存在 → 用去后缀版本
   - Levenshtein 编辑距离 ≤ 3 的最近匹配 → 用最近匹配
   - 都不匹配 → fallback 到 `CircleAlert`
4. 返回所有需要修复的 `LucideIconFix[]`

## 3. 替换函数

### 3.1 applyLucideIconFixes

新增于 `lib/extract-code.ts`：

```typescript
function applyLucideIconFixes(
  files: Record<string, string>,
  fixes: readonly LucideIconFix[]
): void
```

对每个 fix，在对应文件中做全词正则替换（`\bOriginal\b` → `Replacement`），同时覆盖：
- import 声明：`import { CalculatorIcon } from "lucide-react"` → `import { Calculator } from "lucide-react"`
- JSX 使用：`<CalculatorIcon />` → `<Calculator />`
- 变量引用：`{CalculatorIcon}` → `{Calculator}`

原地修改 `files` 对象（与现有后处理函数模式一致）。

## 4. 集成位置

在 `chat-area.tsx` 后处理管线中，插在 `checkImportExportConsistency` 之前：

```
merge files
→ findMissingLocalImportsWithNames (缺失文件补全)
→ checkUndefinedLucideIcons + applyLucideIconFixes (图标名修正) ← 新增
→ checkImportExportConsistency (named/default 不匹配)
→ checkDisallowedImports (禁止包引用)
→ buildSandpackConfig
```

行为：
- 调用 `checkUndefinedLucideIcons(allCompletedFiles)` 获取修复列表
- 如果有修复项，调用 `applyLucideIconFixes(allCompletedFiles, fixes)` 原地替换
- 无 API 调用、无文件数限制、无异步操作、零 token 消耗

## 6. 不做的事

- **不做通用 undefined reference 检测** — 需要 AST parser，复杂度高，留给方案 D（Sandpack 运行时自修复）
- **不调 LLM 修复** — 纯静态替换
- **不改 prompt 层** — 后处理兜底比 prompt 约束更可靠
